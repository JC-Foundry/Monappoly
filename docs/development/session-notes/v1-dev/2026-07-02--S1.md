## 1. 🌟 Triple-card combination bugs (#22, #24) — record-then-apply

The interactive **`MP.GameEngine.Harness`** console app (drives the real `PlayerTurnOrchestrator` through full
turns with **console-answered prompts**, seeds via the real `CardService`) is the vehicle — a menu-driven suite
of **10 card *combination* scenarios** (genuine/no-bonus/doubled/multiply/redirect/downgrade/convert × cancel,
plus immunity-vs-cancel). Used to reproduce and then re-verify each fix.

- **Root cause:** the triple bonus was applied **per card** (`ApplyTripleBonus` called inside each modifier
  card), so a cancel arriving after a drawn modifier double-counted — **double payout + double accumulator
  increment** (#24), and a cancel could fire against an already-downgraded triple (#22).
- **Fix — record-then-apply:** new **`GameModel.TripleBonusModifier`** (factor / recipient / cancelled). Cards
  now **record** onto it (`DiceActionService.ModifyTripleBonus`); the bonus is applied **once** after the
  trigger window (`PlayerService.ResolveTripleBonus`).
- **New `OnTripleBonus` CardTrigger** (John set it to bit `1<<7`, swapping `OnOtherRollsTriple` to `1<<8`, so the
  existing cancel-card JSON `"Trigger": 128` maps to it with **no JSON change**). Separates *rolled a triple*
  from *resolving the bonus*. Wired `CardTriggerService` → `PlayerService` → `PlayerTurnOrchestrator`.
- **Unified the three triple paths** (rolled / held-convert / drawn-Double-upgrade) through a single **`TripleRoll`**
  site so the bonus resolves in exactly one place.
- **Precedence** confirmed by scenario: **cancel beats a drawn modifier** (multiply/redirect), **immunity beats
  cancel** (`CheckCancelledTripleBonusImmunity`). Multiply/redirect still roll their die/dice-off before a cancel
  overrides — a wasted-roll UX quirk inherent to draw-before-cancel ordering, not a correctness bug.

## 2. 🌟 #23 — jail-swap card didn't swap the leave-fees

Card *"Swap places with any other player in jail. Your jail fees to leave are also swapped."*
(`justVisiting.json`) = a Movement **swap** + a Jail **SwapLeaveFee** in one group.

- **Root cause:** `SwapLeaveFee` swaps `player.JailCost` with **`context.ContextPlayerId`** (the partner the
  Movement swap stashes) — but a **resolve-on-draw** card is passed **`context = null`**, so the stash is dropped
  and the fee-swap finds nobody. Positions swapped; fees didn't.
- **Fix (one line, broad reach):** `CardService.ResolveCard` now mints a context for the group —
  **`context ??= new CardActionContext()`** — so a group's actions can share state. Fixes **every**
  resolve-on-draw multi-action card that passes a partner between actions (the jail-swap fee, and the GO
  *"both players receive £200"* swap).
- New harness **scenario 11** (jail) reproduced the miss (alice £50/bob £113 after swap) → after fix, fees swap
  (alice £113/bob £50).

## 9. Decisions locked

  (swap-partner, dice-off winner) communicate — even resolve-on-draw.
- Registration requires **13+ and Terms/Privacy acceptance**.
